SCADA — Summary Statistics

About this dataset — PUBLIC_NEXT_DAY_FPPMW

In the PUBLIC_NEXT_DAY_FPPMW (FPP Daily) data files, these four columns define the exact timing and status of the 4-second SCADA power measurements for batteries and generators.

Column Descriptions

INTERVAL_DATETIME The target timestamp for the 4-second interval. Follows the format YYYY/MM/DD HH:MM:SS and represents the standard grid slot (e.g., :00, :04, :08) the measurement is assigned to.
MEASUREMENT_DATETIME The actual timestamp when the SCADA reading was recorded by AEMO's systems. May differ slightly from INTERVAL_DATETIME due to communication latency or jitter; AEMO snaps each reading to the nearest 4-second slot.
MEASURED_MW The instantaneous power output of the asset in Megawatts (MW) at that specific time. For batteries, positive values denote discharging (generation) and negative values denote charging (load).
MW_QUALITY_FLAG A numeric code indicating data health. Tells you whether MEASURED_MW is a live reading or a substitute inserted because the real data was missing or erroneous (see below).

MW_QUALITY_FLAG Values

The quality flag is critical for ensuring you aren't analysing "frozen" or "bad" data. According to the AEMO FPP Factor Calculation Guide, the digits translate as follows:

0 — Good Data is valid and received directly from the site telemetry.
1 — Substituted / Estimated The raw data was missing. AEMO has "held" the previous known-good value or interpolated a value to fill the gap.
2 — Bad Telemetry is flagged as bad (e.g., communication failure, hardware fault). These points are typically excluded from high-fidelity performance analysis.
3 — Manual Override An AEMO operator has manually adjusted the value to correct for a known system-wide error.

Data Quality

Quality flags are sourced directly from the AEMO FPP dataset. Good (0) = reliable measurement; Suspect (1) = possible communication issue; Bad (2) = sustained failure or substituted value.

Dispatch Energy — Summary Statistics

About this dataset — DISPATCH_UNIT_SOLUTION

Extracted from AEMO's DISPATCH_UNIT_SOLUTION table and published the following day via NEMWEB. Data is fetched live from: Current directory (individual daily files, 11 Feb 2025 – yesterday) or the Archive directory (monthly bundles for older dates, accessed via HTTP Range requests).

INITIALMW Actual power output (MW) at the exact start of the 5-minute dispatch interval. Used as the baseline for the next dispatch instruction.
INITIAL_ENERGY_STORAGE State of Energy (SoE) at the start of the interval (MWh). Represents how much energy was physically available in the battery before the current interval's dispatch began.
ENERGY_STORAGE Target or projected energy level at the end of the interval (MWh). Accounts for the expected charging or discharging activity scheduled during that 5-minute block.

About This App

This tool provides access to two datasets published by the Australian Energy Market Operator (AEMO) via NEMWEB:

4-second SCADA data — from NEMWEB FPPDAILY, available from 28 February 2025 when the Frequency Performance Payments (FPP) scheme commenced non-financial operation.

5-minute dispatch energy data — from NEMWEB Next_Day_Dispatch, available from 11 February 2025. Provides initial energy storage (MWh) and initial MW dispatch for each 5-minute interval.

Data is fetched live from AEMO on each request — no data is stored on this server. Maximum request window: 1 day per query.

SCADA data: AEMO NEMWEB FPPDAILY  |  Energy data: AEMO Next_Day_Dispatch  |  BESS list: AEMO NEM Generation Information  |  Not affiliated with AEMO.
